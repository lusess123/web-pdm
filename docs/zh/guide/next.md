# G6 实践与性能优化

大型 ER 图会同时考验布局、渲染和交互。当前实现基于 G6 5.1.1，主要采用以下策略：

- 字段关系使用具名端口，连线可以准确落到字段左右两侧；
- 有向结构使用 AntV Dagre 层次布局，关系探索使用 Force 布局；
- 普通数据更新保留节点位置，只有结构变化时才重新布局；
- 节点会按缩放比例减少细节，避免在小比例下渲染无意义文字；
- 发布包启用 Worker 前，需要先验证消费端能够正确加载 Worker 资源；
- 所有 G6 异步操作由运行时适配层串行调度，组件卸载时安全结束；
- Minimap、主题、适应画布和 PNG 导出都使用 G6 5 的原生 API。

性能优化需要用接近真实业务的数据验证。至少应覆盖空图、稀疏关系图、高连接度图，以及桌面和移动端视口；不要只根据少量演示数据判断效果。
